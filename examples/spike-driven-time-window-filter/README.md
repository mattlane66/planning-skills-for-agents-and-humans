# Spike-Driven Time Window Filter — Step-by-Step Example

This example shows how to use the planning skills when a promising shape still contains a real unknown.

> Add natural-language time window input to an existing analytics filter without pretending the parser behavior is already understood.

The point is to show:
- a flagged unknown (`⚠️`)
- a standalone spike file
- refinement of the shape after the spike
- breadboarding only after the mechanism becomes concrete
