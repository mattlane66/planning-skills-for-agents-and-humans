# Brownfield Admin Search — Step-by-Step Example

This example shows how to use the planning skills on an existing system instead of a greenfield feature.

> The current admin page already lists users. The new goal is to add search and keep the current behavior that already works.

The point of this example is to show:
- how to use `CURRENT`
- how to shape against an existing baseline
- how to breadboard existing and new behavior together

## Files in this example

- `00-current-notes.md` — notes about the current system and desired change
- `01-frame.md` — frame for the change
- `02-shaping.md` — shaping with `CURRENT` plus a proposed direction
- `03-breadboard.md` — breadboard showing existing + new behavior together

## What to notice

- `CURRENT` is treated as a real baseline, not as a throwaway note.
- The shaped change is described against that baseline.
- The breadboard includes both what remains and what changes.
